# KelpyShark Package Manager

The KelpyShark package manager handles library versioning, dependency resolution,
installation, and publishing. It is inspired by npm, pip, and Cargo.

---

## Quick Start

```bash
# Create a new project
kelpy new my_project
cd my_project

# Install a package
kelpy install http

# Install all dependencies from kelpy.toml
kelpy install

# Publish your library
kelpy publish
```

---

## Project Layout

Every KelpyShark project follows this structure:

```
my_project/
в”ңв”Җв”Җ kelpy.toml          # Package manifest
в”ңв”Җв”Җ kelpy.lock          # Pinned dependency versions (auto-generated)
в”ңв”Җв”Җ src/
в”Ӯ   в””в”Җв”Җ main.ks         # Entry point
в””в”Җв”Җ libs/               # Installed dependencies
    в”ңв”Җв”Җ http/
    в””в”Җв”Җ json/
```

---

## `kelpy.toml` вҖ” Package Manifest

The manifest file describes your project and its dependencies.

```toml
[package]
name = "my_project"
version = "0.1.0"
description = "A cool KelpyShark project"

[dependencies]
http = "1.0.0"
json = "0.2.1"
```

### `[package]` Section

| Field | Required | Description |
|---|---|---|
| `name` | Yes | Package name (lowercase, hyphens allowed) |
| `version` | Yes | Semantic version (`MAJOR.MINOR.PATCH`) |
| `description` | No | Human-readable description |

### `[dependencies]` Section

Each entry maps a package name to a version string:

```toml
[dependencies]
math_extra = "1.0.0"
```

---

## `kelpy.lock` вҖ” Lock File

When you run `kelpy install`, a `kelpy.lock` file is generated that pins the exact
resolved versions of every transitive dependency. This ensures reproducible builds.

```
# Auto-generated by KelpyShark package manager
# Do not edit manually

json=0.2.1
http=1.0.0
```

**Commit this file** to version control so collaborators get identical dependency versions.

## CLI Commands

### `kelpy install [package]`

Install dependencies.

```bash
# Install all dependencies from kelpy.toml
kelpy install

# Install a specific package
kelpy install http
```

When installing a specific package, it is added to `kelpy.toml` and fetched from the
registry into the `libs/` directory.

### `kelpy publish`

Publish your library to the registry so others can install it.

```bash
kelpy publish
```

Reads `kelpy.toml` for metadata and packages the `src/` directory into a `.kslib`
archive in the registry.

### `kelpy new <name>`

Scaffold a new KelpyShark project:

```bash
kelpy new hello_world
```

Creates:

```
hello_world/
в”ңв”Җв”Җ kelpy.toml
в””в”Җв”Җ src/
    в””в”Җв”Җ main.ks
```

## Dependency Resolution

The resolver uses **topological sorting** (Kahn's algorithm) to determine a safe
installation order. It guarantees:

1. Every dependency is installed before any package that depends on it.
2. Circular dependencies are detected and reported as errors.
3. Transitive dependencies (dependencies of dependencies) are resolved automatically.

### Example

Given these dependency relationships:

```
my_project вҶ’ http (1.0.0)
my_project вҶ’ json (0.2.1)
http       вҶ’ json (0.2.1)
```

The resolver produces the install order: `json`, then `http`.

## Registry

The package registry stores published `.kslib` packages. By default, packages
are stored in the user-local registry at:

| Platform | Location |
|---|---|
| Linux / macOS | `~/.kelpyshark/registry/` |
| Windows | `%USERPROFILE%\.kelpyshark\registry\` |

### Registry Layout

```
~/.kelpyshark/registry/
в”ңв”Җв”Җ http/
в”Ӯ   в””в”Җв”Җ 1.0.0/
в”Ӯ       в”ңв”Җв”Җ kelpy.toml
в”Ӯ       в””в”Җв”Җ src/
в”Ӯ           в””в”Җв”Җ lib.ks
в””в”Җв”Җ json/
    в””в”Җв”Җ 0.2.1/
        в”ңв”Җв”Җ kelpy.toml
        в””в”Җв”Җ src/
            в””в”Җв”Җ lib.ks
```

### Publishing a Package

When `kelpy publish` is run:

1. The manifest is read from `kelpy.toml`.
2. The source files under `src/` are copied into the registry.
3. A registry entry is created at `~/.kelpyshark/registry/<name>/<version>/`.

## Library Format

KelpyShark libraries use the `.kslib` extension and follow this structure:

```
my_library.kslib/
в”ңв”Җв”Җ kelpy.toml
в””в”Җв”Җ src/
    в”ңв”Җв”Җ lib.ks          # Library entry point
    в””в”Җв”Җ helpers.ks      # Additional modules
```

The `lib.ks` file is the entry point loaded by `import my_library`.

---

## Using Installed Packages

After installation, use the `import` statement in your code:

```
import http
import json

data = json.parse('{"name": "KelpyShark"}')
print data["name"]
```

Imports search the following locations in order:

1. Standard library modules (`math`, `strings`, `io`, `json`, `sys`)
2. Project `libs/` directory
3. User-local registry

## Version Policy

KelpyShark uses **Semantic Versioning** (SemVer):

| Component | Meaning |
|---|---|
| **MAJOR** | Breaking changes |
| **MINOR** | New features, backward compatible |
| **PATCH** | Bug fixes, backward compatible |

Example: `1.2.3` вҶ’ major=1, minor=2, patch=3

## Troubleshooting

### "Package not found in registry"

The package hasn't been published yet. Ask the author to run `kelpy publish`, or
check the spelling of the package name.

### "Circular dependency detected"

Two or more packages depend on each other, forming a cycle. Restructure the
libraries to break the cycle.

### "Could not read kelpy.toml"

Make sure you're running commands from the project root directory and that
`kelpy.toml` exists with valid syntax.
